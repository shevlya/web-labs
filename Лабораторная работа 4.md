## Способы аутентификации
На выбор вам предлагается два варианта для выполнения.

### Базовый вариант выыполнения
В конфигурационном файле включите авторизацию Basic / Form. Продемонстрируйте оба способа в postman. Объясните, как передаются данные пользователя в каждом способе.

Включите логирование Security, передав debug=true в аннотацию EnableWebSecurity. Проследите цепочку фильтров, вызывающихся при выполнении запроса. Объясните назначение фильтров.

### Продвинутый вариант выполнения

В данной части лабораторной необходимо реализовать **механизм токенов авторизации вручную**, без использования готовых библиотек JWT.
Формат токена в данной лабораторной упрощён и имеет вид:

```
<base64Url(payload)>.<base64Url(signature)>
```

---

#### Подготовка

**Секретный ключ**

Для подписи токенов используется секретный ключ.

1. Придумайте случайную строку длиной **не менее 32 символов**.

2. Сохраните ключ в переменную среды `JWT_SECRET`.

3. В приложении получайте ключ через:

```java
String secret = System.getenv("JWT_SECRET");
```

Секретный ключ **нельзя хранить в коде**.

#### Генерация токенов при логине
Добавьте эндпоинт `/auth/login`.
После успешной проверки логина и пароля в эндпоинте необходимо создать:

* **Access Token** — срок действия 15 минут
* **Refresh Token** — срок действия 7 дней

Токены должны возвращаться клиенту в ответе.

#### Структура данных токена

Токен состоит из **нагрузки** — набора данных о пользователе.

Для удобства можно использовать `Map<String, Object>`.

#### Access Token

Нагрузка должна содержать:

* `userId` — идентификатор пользователя, которому принадлежит токен
* `roles` — список ролей пользователя
* `iat` — время создания токена (UNIX TIME)
* `exp` — время истечения токена (UNIX TIME)

Пример:

```json
{
  "userId": 1,
  "roles": ["USER"],
  "iat": 1700000000,
  "exp": 1700000900
}
```

#### Refresh Token

Payload содержит:

* `userId`
* `iat`
* `exp`

Роли в refresh token добавлять не нужно.

#### Алгоритм создания токена

Создайте отдельный сервис, например:

```
TokenService
```

В этом сервисе реализуйте метод:

```
generateToken(Map<String, Object> payload)
```

Далее выполните следующие шаги.

**Шаг 1. Преобразование payload в JSON**

Используйте Jackson для преобразования в JSON-строку:

```java
ObjectMapper mapper = new ObjectMapper();
String json = mapper.writeValueAsString(payload);
```

**Шаг 2. Кодирование payload**

Закодируйте JSON в Base64 URL.

```java
String encodedPayload =
        Base64.getUrlEncoder()
              .withoutPadding()
              .encodeToString(json.getBytes());
```

**Шаг 3. Создание подписи**

Подпись создаётся с использованием алгоритма **HmacSHA256**.

1. Создайте объект `Mac`
2. Инициализируйте его секретным ключом
3. Подпишите строку `encodedPayload`

Пример:

```java
Mac mac = Mac.getInstance("HmacSHA256");

SecretKeySpec key =
        new SecretKeySpec(secret.getBytes(), "HmacSHA256");

mac.init(key);

byte[] signatureBytes =
        mac.doFinal(encodedPayload.getBytes());
```

**Шаг 4. Кодирование подписи**

Подпись также нужно закодировать в Base64 URL:

```java
String encodedSignature =
        Base64.getUrlEncoder()
              .withoutPadding()
              .encodeToString(signatureBytes);
```


**Шаг 5. Сборка токена**

Токен собирается как строка:

```
payload.signature
```

#### Проверка токена

Необходимо реализовать фильтр:

```
JwtFilter
```

Фильтр должен наследоваться от:

```
OncePerRequestFilter
```


#### Исключения из фильтра

Фильтр не должен применяться к:

```
/auth/login
/auth/refresh
/users/register
```

#### Извлечение токена

Токен передаётся в заголовке:

```
Authorization: Bearer <token>
```

Необходимо:

1. получить заголовок `Authorization`
2. удалить префикс `Bearer `
3. получить строку токена

#### Проверка структуры

Разделите токен по `"."`:

Должно быть **ровно две части**:

* payload
* signature

#### Проверка подписи

1. Возьмите первую часть токена (payload)
2. Снова создайте подпись тем же алгоритмом
3. Сравните полученную подпись с подписью из токена

Если подписи не совпадают:

```
HTTP 401 Unauthorized
```

#### Проверка срока действия

1. Декодируйте payload:

```java
byte[] decoded =
        Base64.getUrlDecoder().decode(payloadPart);
```

2. Преобразуйте JSON обратно в Map через Jackson.

3. Получите поле `exp`.

4. Сравните с текущим временем.

Если токен истёк:

```
HTTP 401 Unauthorized
```

#### Установка пользователя в SecurityContext

Если токен корректный:

1. извлеките `userId` и `roles`
2. создайте объект `Authentication`
3. сохраните его в `SecurityContextHolder`

После этого пользователь считается авторизованным.

#### Обновление Access Token

Создайте эндпоинт:

```
POST /auth/refresh
```

Он должен:

1. принять Refresh Token
2. проверить подпись
3. проверить срок действия
4. извлечь `userId`
5. создать новый Access Token
6. вернуть его клиенту

#### Ошибки

Если Refresh Token:

* повреждён
* подпись неверна
* срок истёк

необходимо вернуть:

```
HTTP 401 Unauthorized
```
