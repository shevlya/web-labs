# Лабораторная работа №5

Ниже представлена краткая инструкция по настройке окружения и созданию нового проекта:  
1. Установите LTS-версию [Node.js](https://nodejs.org/en/download/).  
2. В терминале выполните команду для установки Angular CLI:  
   ```sh
   npm install -g @angular/cli
   ```
3. Перейдите в каталог, в котором планируется размещение проекта, и создайте новый Angular-проект:  
   ```sh
   ng new <имя-проекта>
   ```
4. Перейдите в созданную директорию проекта. Для запуска приложения выполните команду:  
   ```sh
   ng serve --open
   ```
   После успешного запуска приложение будет доступно по адресу `http://localhost:4200/`.  
5. При использовании **Visual Studio Code** рекомендуется установить [Angular Language Service](https://marketplace.visualstudio.com/items?itemName=Angular.ng-template) для улучшенной поддержки Angular-шаблонов.

Создайте следующие компоненты с помощью Angular CLI:  
- **TaskListComponent** — отображает список задач.  
- **EditTaskComponent** — форма для редактирования/создания задачи.
- **LoginComponent** - компонент входа в систему.  

Настройте маршрутизацию, добавив соответствующие пути:  
- `/tasks` — список всех задач текущего пользователя.  
- `/tasks/{taskId}` — редактирование задачи с идентификатором `taskId`.  
- `/tasks/new` — создание новой задачи.  
- `/login` — вход в систему.

## Auth
Создайте файл auth.interceptor.ts. Интерсептор получает сохранённые учётные данные и добавляет заголовок Authorization к каждому запросу. Для Basic/Token вариантов реализаций механизм схож.
```typescript
export const authInterceptor: HttpInterceptorFn = (req, next) => {
  const credentials = ...;

  if (credentials) {
    const authReq = req.clone({
      headers: req.headers.set('Authorization', `Basic ${credentials}`)
    });
    return next(authReq);
  }

  return next(req);
};
```

## Компонент логина
На странице логина два поля: 
- логин (input type="text")
- пароль (input type="password")

Отправляйте запрос на /auth/login. В случае прихода 401 сигнализируйте пользователю, что он ввел неправильную информацию.
Для варианта с Basic Auth можно выполнять запрос на `/auth/me` и также анализировать статус.
Если ответ от сервера успешный, то сохраните в sessionStorage информацию о логине и пароле пользователя (или о токене) и сделайте редирект пользователя на страницу задач.
В `auth.interceptor` используйте сохраненную в sessionStorage информацию. 

## **Создание компонента отображения задачи**  
1. Создайте компонент, который будет отображать сводную информацию о задаче. В качестве входного параметра данный компонент принимает DTO задачи.
2. Для каждой задачи отобразите: название, статус (каждый статус идентифицируйте цветом), дата и время создания.
3. Добавьте кнопку удаления. Добавьте кнопке удаления обработчик. В обработчике вызывайте событие deleteTask, передав в него идентификатор задачи.
4. Форматирование даты выполняется с помощью [DatePipe](https://angular.dev/guide/pipes/template#chaining-pipes).  
5. При нажатии на кнопку "Редактировать" пользователь перенаправляется на страницу редактирования задачи (`/tasks/{taskId}`) с использованием [RouterLink](https://angular.io/api/router/RouterLink).  
6. Проверьте работоспособность вашего компонента, добавив его в разметку app.component.html (после отладки этот шаг можно убрать)

## **Отображение списка задач**  
1. Реализуйте загрузку данных с сервера.
2. Добавьте кнопку "Создать задачу", перенаправляющую пользователя на `/tasks/new`.
3. Добавьте отрисовку списка задач через цикл @for. Передавайте соответствующую задачу.
4. Подпишитесь на событие deleteTask. В обработчике события вызовите API удаления задачи с сервера и, в случае успеха, удаляйте задачу из массива задач.

## **Создание и редактирование задач**  
1. Реализуйте форму редактирования/создания задачи с полями:  
   - `input type="text"` — название;
   - select - статус задачи.
2. Используйте **Template-Driven Forms** или **Reactive Forms** для привязки данных.  
3. При редактировании загружайте данные задачи по её идентификатору.  
4. После сохранения пользователь должен быть перенаправлен на `/tasks`.  
5. Опционально: добавить валидацию формы.
